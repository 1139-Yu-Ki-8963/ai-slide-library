# スライド蓄積簿

公開スライドカタログ（index.html）の元データ。1 行 = 1 スライド。カタログは scripts/build-catalog.mjs がこの表から再生成する。

## 記入規則

- スライドキーは連番禁止。内容を要約した意味語で付ける（番号からは情報を得られないため）
- 1スライド = 1フォルダ（`slides/<スライドキー>/解説スライド.html`）。フォルダごと配布できる状態を保つ
- 各フォルダに `検査記録.md` を残す（観点レビュー表の照合結果と、そのスライド限定の観点。更新時の再検査項目になる）
- 公開リポジトリのため、特定環境の絶対パス・特定顧客の情報はスライドにも本表にも含めない
- 対象ツール・仕組み・テーマ・導入段階は下記のタグ語彙一覧から選ぶ。複数値は「、」区切り、該当なしは「—」と書く
- 語彙の表記統一: 対象ツール・仕組み（技術要素）は正式名称の英語表記、テーマ（読み手の悩み）・導入段階は日本語。1語 = 1概念とし複合語を作らない
- タグ語彙一覧に無い語を表で使うとビルドがエラーで止まる。既存語で表せない場合のみ、先に語彙一覧へ追加してから使う（表記ゆれで検索が壊れるのを防ぐため）
- 該当スライドがまだ無い語も語彙一覧に事前登録してよい。カタログでは0枚の語は選択不可のグレー表示になる
- 行を追加・変更したら `node scripts/build-catalog.mjs` を実行して index.html を再生成する
- slides/ 配下のフォルダと本表の行は 1:1。本表に無いフォルダ、フォルダの無い行のどちらもビルドがエラーで止まる（掲載漏れ・幽霊エントリの機械防止）
- 「導入段階」列は、AI駆動開発導入の整備順序を示す六区分（全体像・現状診断・土台整備・ツール導入と安全装置・開発フロー確立・拡大・継続改善。全体像は五段階全体を俯瞰するメタ区分）のうちどの区分の説明に位置づくかを示す。値は語彙一覧の導入段階軸から選ぶ。複数値は「、」区切り可、該当なしは「—」
- 「取り込み実体」列は、この仕組みを構成する実体を `<種別>: <内容の要約名>` の形式で「、」区切りで記入する。種別は hook / 規約 / スクリプト / 辞書 / テンプレート / 運用手順 の 6 語に限定する
- 「パック」列は改善パック（実体ファイルの zip 配布）の有無。現在は全て「—」（配布物は内容承認後に追加予定）。値は「あり」または「—」
- 「導入プロンプト」列は、カタログの「導入する」ボタン（プロンプト表示）を出すかの指定。値は「あり」または「—」。「あり」の行は `slides/<スライドキー>/取り込みプロンプト.md` が必須（無いとビルドエラー）。現在は全て「—」（ユーザーが指定したスライドのみ「あり」にする）
