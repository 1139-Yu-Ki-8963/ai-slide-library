CLAUDE CODE — OUTPUT STYLE

Output Style — AI の応答態度を場面に合わせて切り替える機能

同じ指示でも Style によって「すぐ動く」か「確認してから動く」かが変わる

Proactive Style が有効時に AI が受け取る指示
# Output Style: Proactive # Proactive Style Active The user chose continuous, autonomous execution. You should: 1. Execute immediately Start implementing right away. Make reasonable assumptions. 2. Minimize interruptions Prefer assumptions over asking questions. 3. Prefer action over planning Do not enter plan mode unless explicitly asked. 4. Expect course corrections 5. Do not take destructive actions Deletions and shared-system changes still need confirmation.
有効

Proactive（自律型）

即座に実行を開始し、妥当な判断ができる場面では確認せずに進む。方向転換は通常の入力として受け止める。ただしデータ削除・共有システムへの操作は必ず確認する。

向いている場面: 方針が明確で大量の作業を一気に進めたいとき

既定

Normal（標準型）

低リスクな操作は即実行、判断が必要な場面で確認する。大きな変更の前に計画の概要を提示し、不明点があれば事前に質問する。

向いている場面: 初めての作業や、判断の余地が大きいとき

学習

Pedagogical（解説型）

各ステップで「なぜそうするか」を添えて説明する。複数の方法がある場面で比較を提示し、コードの背景知識や設計判断の根拠を補足する。

向いている場面: 仕組みを理解しながら進めたいとき
