Claude Code — AI駆動開発の docs 配置ガイド

.md に書いてここに置く — それだけで AI の行動が変わる

書く内容 置く場所 AI への効果
プロジェクト概要 技術スタック・用語集・設定索引 CLAUDE.md ルート直下（親ディレクトリから順に読込）
プレーン Markdown・frontmatter 不要 		毎回のセッション開始時に自動で読み込まれる
→ AI が常にプロジェクトの文脈を把握した状態で作業する
コーディング規約 命名ルール・DB設計規約・レビュー基準 .claude/rules/*.md プレーン Markdown（frontmatter 不要）
00-global.md   常時適用
10-frontend.md   paths: src/**
50-database.md   paths: db/** 		paths 条件で該当ファイル編集時だけ注入される
→ 生成されるコードが規約に従う
操作手順・ワークフロー デプロイ手順・コミットフロー .claude/skills/ SKILL.md に手順を記述
adapt/   safe-commit/ 		/コマンド名 で必要な時だけ呼び出される
→ 手順が属人化せず再現可能になる
サブエージェント定義 専門役割の分離・タスク委任 .claude/agents/*.md YAML frontmatter で model/tools を指定
reviewer.md   investigator.md 		Agent ツールから専門エージェントとして呼び出される
→ 役割ごとにツール権限とモデルを制御できる
実行制御・環境設定 フック・権限・外部ツール接続 settings.json + .mcp.json settings.json   hooks / permissions
.mcp.json   MCP サーバー接続
allow / deny でコマンド許可を制御 		PreToolUse / PostToolUse 等のフックで動作を制御
→ コマンド実行の許可・拒否を精密に管理できる
