AI開発ガバナンス — 機密情報保護

機密情報は「コミット・push・公開」の三重の関門で機械的に止める

AI がうっかり API キーや顧客情報を書き込んでも、履歴に残る前・外に出る前・公開する前の各段階で hook（操作に自動で割り込む仕組み）が検出して停止する。

1

第一関門: コミット時の検査

コミット対象の追加行に API トークン・秘密鍵らしき文字列があれば、コミット自体をエラー終了で停止する。検出時は件数とファイル名のみ通知し、機密値そのものは表示しない。

API トークン 秘密鍵 認証情報ファイル
第一関門を通過した変更も、外部リポジトリへ出る直前に再検査される
2

第二関門: push 時の再検査

push 対象の未送信コミット全体の追加行を再走査し、すり抜けた機密値をリポジトリの外へ出る直前で止める。検出時は履歴からの除去までを要求する。

未 push コミットの全追加行 履歴からの除去を要求
リポジトリを公開・配布する前には、差分ではなく全体の走査が入る
3

最終防衛線: 公開前の機密漏洩走査

公開の前にスナップショット全体を走査し、機密文字列に加えて特定環境の絶対パス・顧客情報の残存まで最終確認する。

全ファイルを対象に走査 絶対パス・顧客情報も検出対象
