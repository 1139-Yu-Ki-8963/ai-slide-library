## 構造・図解の採用評価（5 項目）

### 対比カード 2 枚（compare-card）
1. 導出元: 2 機構が「検出対象の異なる検出器」という比較・対比の性質を持つこと
2. 採用理由: compare-row / compare-card は時間構造を持たない並置比較の既定部品であり、対象の性質（対比）に一致する
3. 理解への寄与: 検出対象・検証者・原理・出力・成立の鍵を同一の行構成で並べ、左右の見比べで相違（弱モデル vs 独立懐疑者、前提の欠落 vs 主張の誤り）を確かめられる
4. 代替棄却: 時系列型（時間経過の説明ではない）・段階図（成熟度の話ではない）・層構造単独（2 機構は並列であり上下関係ではない）を棄却
5. 全構成要素は上記 1〜4 で説明済み。説明できない構造なし

### 検出対象バッジ（detect-row）
1. 導出元: 各機構の「何を検出するか」が本スライドの結論の核であること
2. 採用理由: 警告色トークン（--critical 系）は欠陥・検出という意味役割に対応する
3. 理解への寄与: 両カードの同位置・同形式に置くことで対比の軸を視覚的に固定する
4. 代替棄却: カード見出しへの埋め込み（対比の軸が立たない）を棄却
5. 説明済み

### 機械強制ゲート帯（layer-band）
1. 導出元: 2 検証の実行記録が提示直前のゲートで検査されるという「通過条件」の関係
2. 採用理由: 全幅帯 + 番号チップは通過・集約構造の既定部品
3. 理解への寄与: 2 カードの成果が 1 つのゲートに集約される流れを全幅で示し、例チップ 4 つで発火点と挙動を具体化する
4. 代替棄却: イベント線 + バッジ（時間軸が存在しない図では意味を持たない）を棄却
5. 説明済み

### 読み順チップとフッタ要約（step-chip + steps）
1. 導出元: 読み手が図を追う順序（1 暗黙知抽出 → 2 敵対的検証 → 3 機械強制）
2. 採用理由: 図中チップと下部要約の同番号・同色対応は部品カタログの規定
3. 理解への寄与: 図の読み終わりに 3 項目の要約で結論を回収できる
4. 代替棄却: 番号なし（読み順が不定になる）を棄却
5. 説明済み

## 設計領域の記録（11 領域）
1. 情報設計: 結論（ヘッダ）→ 対比（2 カード × 5 行）→ 統合（ゲート帯）→ 要約（フッタ 3 項目）の 4 分節
2. 構造化: header / article / footer のセマンティック要素、ul.fact-list による属性列挙、h1〜h3 の見出し階層
3. 全体的レイアウト: 1280×720 固定・縦 4 段構成（ヘッダ / 対比 / 帯 / フッタ）
4. 局所的レイアウト: fact-key（固定幅ラベル）+ 本文の 2 カラム行で左右カードの行位置を揃える
5. 視覚設計: 見出し 30px / カード見出し 16px / 本文 12.5px の 3 段階。strong の太字強調は各行 1 箇所程度に抑制
6. カラーリング設計: デザイントークンのみ使用。--critical = 検出対象と機構 2、--claude = 機構 1、--good = ゲート通過。彩度色は意味役割に限定
7. 図解設計: 図解候補のうち比較（対比カード）・通過条件（ゲート帯）・順序（番号チップ）を採用。時系列図・数量図は不採用（対象に時間・数量の性質を持つ関係が存在しないため）
8. 読解導線: 番号チップ 1→2→3 と左 → 右 → 下の配置で視線誘導
9. 操作設計: インタラクション不採用（静的 1 枚スライドであり、配布・投影には操作を要求しない方が適するため）。ウィンドウリサイズ追従の scale スクリプトのみ保持
10. 自己完結性: 外部参照 0 件（CDN・Web フォント・外部画像・fetch なし）
11. 単一 HTML としての成立性: DOCTYPE / html lang="ja" / viewport / charset / title を備えた単一ファイル。CSS / JS はすべてインライン
