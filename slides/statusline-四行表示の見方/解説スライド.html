CLAUDE CODE — STATUSLINE

statusline — 入力欄の下に任意の情報を常時表示できるカスタマイズ機能

この例ではモデル・セッション・リソース消費・フロー進捗の 4 行を設定している

claude — statusline
🤖 Opus 4.6 (1M context) | 📂 my-project | Branch: feature/add-dashboard | my-account
Session: a1b2c3d4 | Name: 機能実装セッション
◔ ctx:23% ◑ 5h:58% ◕ 7d:84%
⚙ Phase 3 実装 ▰▰▰▰▰▱▱▱▱▱▱▱▱ 5/12 API endpoint 追加
1

モデル・作業場所

  • 🤖 モデル名—使用中の Claude モデル（Opus 4.6 / Sonnet 5 等）
  • 📂 フォルダ—カレントディレクトリのベース名
  • Branch—現在の Git ブランチ（git 管理外では非表示）
  • アカウント—接続先の識別名（.claude-xxx → xxx）
2

セッション情報

  • Session—セッション ID の先頭 8 文字
  • Name—/name で設定したセッション名（未設定なら非表示）
3

リソース使用率

  • ctx—コンテキストウィンドウ使用率（会話が長くなると上昇）
  • 5h—5 時間あたりのレートリミット消費率
  • 7d—7 日間あたりのレートリミット消費率
○ 0–12% ◔ 13–37% ◑ 38–62% ◕ 63–87% ● 88%+ <50% 50–79% ≥80% 取得不能
4

開発フロー進捗

  • Phase N 名称—現在のフェーズ番号と名前
  • ▰▱ 進捗バー—13 文字幅の達成度表示
  • N/M—現在ステップ / 総ステップ数
  • ステップ名—進行中のステップの名称

※ flow-status.json が存在する場合のみ表示される条件付き行

出典: statusline.py（settings.json の statusline.command で指定する Python スクリプト）
