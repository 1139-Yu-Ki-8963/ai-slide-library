## タスク限定の観点（更新時に再検査する）

- Permission と Sandbox は別の層で動く。Permission はツール呼び出し制御（Claude Code 内部）、Sandbox は Bash の OS レベル制限（Seatbelt / bubblewrap）
- 2層の適用順: Permission が先、Sandbox が後。矢印で「Permission を通過した Bash コマンドに、さらに Sandbox が適用される」と明示
- Permission の deny は「他の設定で上書きできない最強の制限」であることを明記
- Sandbox の具体例に「認証情報ファイルの読取遮断」を含める（sandbox.credentials 機能の反映）
- WSL はこのスライドの対象外。Windows 環境の説明が必要な場合は別資料で扱う
- 層の色分け: Permission = 青系（--perm）、Sandbox = 緑系（--sandbox）
