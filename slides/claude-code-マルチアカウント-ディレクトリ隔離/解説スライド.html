CLAUDE CODE 環境構築

複数アカウントを並行して使う方法

CLAUDE_CONFIG_DIR による設定ディレクトリの隔離

Bedrock サブ環境の設定

~/.claude-bedrock-{id}/settings.json に認証情報を記載する。env ブロックに CLAUDE_CODE_USE_BEDROCK と AWS_REGION を必ず含める。

命名規則: ~/.claude-bedrock-{識別名}/

ディレクトリ作成: mkdir -p ~/.claude-bedrock-dev

Bearer Token

AccessKey が {user}-at-{accountId} 形式の場合

"env": { "CLAUDE_CODE_USE_BEDROCK": "1", "AWS_REGION": "ap-northeast-1", "AWS_BEARER_TOKEN_BEDROCK": "ABSK..." }

確認: /status で Provider: Amazon Bedrock と表示

IAM アクセスキー

AccessKey が AKIA... で始まる場合

"env": { "CLAUDE_CODE_USE_BEDROCK": "1", "AWS_REGION": "ap-northeast-1", "AWS_ACCESS_KEY_ID": "AKIA...", "AWS_SECRET_ACCESS_KEY": "..." }

確認: /status で Provider: Amazon Bedrock と表示

SSO / Identity Center

aws sso login で一時クレデンシャルを取得

事前に aws sso login --profile {profile} が必要

"env": { "CLAUDE_CODE_USE_BEDROCK": "1", "AWS_REGION": "ap-northeast-1", "AWS_PROFILE": "my-sso-profile" }

確認: /status で Provider: Amazon Bedrock と表示

モデル版固定と制約

Bedrock のデフォルトモデルは Anthropic API より遅延する。settings.json の env にリージョンプレフィックス付きモデル ID を追加して固定する（例: jp.anthropic.claude-opus-4-8）。

WebSearch ツールは利用不可。

us-east-1 → us.anthropic. ap-northeast-1 → jp.anthropic. eu-west-1 → eu.anthropic.
