REPOSITORY STRUCTURE — DUAL AI

Cursor と Claude Code は設定フォルダが分離——1 リポジトリで共存する

.cursor/ と .claude/ は独立——設計書・可視化は project-portal/ に集約して共有する

■ Cursor の設定は .cursor/ に、Claude Code の設定は .claude/ に閉じるため互いに干渉しない

■ 設計書も可視化サイトも project-portal/ に集約し、両方の AI が共通で参照する

フォルダ構成 — Cursor + Claude Code 共存時
<repo>/
├─ AGENTS.mdCursor 自動読込
├─ CLAUDE.mdClaude Code 自動読込
│
├─ project-portal/両 AI が参照
│  ├─ 基盤情報/ 技術スタック・用語辞書
│  ├─ 規約/ コーディング規約・命名規約
│  ├─ 一覧・設計図/ 画面・API・テーブル等 9 種
│  └─ 交差ビュー/ 権限マトリクス・CRUD 図
│
├─ .cursor/ Cursor 専用
│  ├─ rules/*.mdc ファイル条件で適用
│  └─ skills/*.md 名前で呼び出し
│
└─ .claude/ Claude Code 専用
    ├─ rules/
    │  ├─ always/ 常時注入
    │  └─ scoped/ 条件注入
    └─ skills/*.md /コマンドで呼び出し
1

Cursor の設定領域

AGENTS.md を自動読込。.cursor/rules/ の MDC 規約はファイル条件で自動適用。.cursor/skills/ に手順書を配置し名前で呼び出す

対象: AGENTS.md / .cursor/rules/*.mdc / .cursor/skills/*.md

2

project-portal/ — 共有資産

設計情報と技術資産を一元管理するポータル。規約・一覧・交差ビュー・解析概要を HTML サイトに集約。人も AI もブラウザで同じ情報を参照する

対象: project-portal/ 配下（基盤情報・規約・一覧 9 種・交差ビュー 4 種・解析概要）

3

Claude Code の設定領域

CLAUDE.md を自動読込。.claude/rules/ の always/ は常時注入、scoped/ は条件注入。.claude/skills/ に手順書を配置し /コマンドで呼び出す

対象: CLAUDE.md / .claude/rules/{always,scoped}/ / .claude/skills/
