CODEX CLI

AI駆動開発の docs 配置ガイド

AGENTS.md と .codex/ に書いてここに置く — それだけで AI の行動が変わる

書く内容 置く場所 AI への効果
プロジェクト指示 技術スタック・用語集・作業規約・避けるべきパス AGENTS.md ルート〜サブディレクトリに階層配置
近い階層ほど優先・全ファイルを連結読込
Cursor/Copilot と共有可 		セッション開始時にルートから順に自動で読み込まれる
→ ディレクトリ階層がそのままスコープ付きルールになる
中央設定 モデル選択・MCP接続・サンドボックス・承認ポリシー .codex/config.toml TOML 形式（信頼済みプロジェクトのみ読込）
sandbox_mode   approval_policy
[mcp_servers.名前]   外部ツール接続 		モデル・権限・外部ツールを一元管理できる
→ sandbox で read-only / write / full-access を制御する
操作手順・ワークフロー デプロイ手順・コミットフロー .agents/skills/ SKILL.md + scripts/ + references/
deploy-flow/   safe-commit/ 		/skills や $ で必要な時だけ呼び出される
→ 手順が属人化せず再現可能になる
サブエージェント定義 専門役割の分離・タスク委任 .codex/agents/*.toml TOML 形式で name/description/model を指定
verifier.toml   reviewer.toml 		独自コンテキストで最大6並列で実行される
→ 複雑なタスクを専門エージェントに自動委任できる
実行フック ツール実行前後の自動処理 .codex/hooks.json JSON 形式・stdin/stdout で JSON をやり取り
PreToolUse   PostToolUse   Stop 		ツール実行やセッション終了時にスクリプトが自動発火
→ lint・テスト・通知などを自動化できる
