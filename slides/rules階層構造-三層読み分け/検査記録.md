## タスク限定の観点

| 観点 | 内容 |
|---|---|
| 環境固有データの排除 | 規約数（19件・18件等）のようなPC固有の値を含めない。構造パターンの例示に留める |
| ツリーの深さ | scope → 分野 → 規約名 → ファイルの 4 階層（+ ルートで 5 階層）を表示すること |
| 例示ファイルの汎用性 | check-*.sh, design-notes.txt, naming-values.txt, prh.yml は Claude Code の rules 構造で一般的なパターンとして使用 |
