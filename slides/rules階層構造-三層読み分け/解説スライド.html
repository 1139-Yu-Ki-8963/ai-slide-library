CLAUDE CODE 設定管理

規約は「所有レベル」×「注入タイミング」の 2 軸で整理する

まずグローバルとプロジェクト固有に分け、それぞれの中で always / scoped を使い分ける

グローバル ~/agent-home/rules/ — 全プロジェクト共通の規約
always/ 全セッションで常時注入
├─ naming/commit-branch/ 命名規約 hook 同居
├─ review-checklist/text-dictionary/ 語彙辞書
├─ agent/subagent-selection/ 委任規約 hook 同居
└─ response/language/ 応答言語・文体
scoped/ 対象パス操作時のみ注入
├─ review-checklist/code/ paths: **/*.ts
├─ dev-flow/gate/ paths: src/**
└─ agent-config/hooks/ paths: tools/hooks/**
値の委譲（追加のみ） — 語彙・命名値をプロジェクトで追加できる。枠組みの上書きは不可
プロジェクト固有 <repo>/.claude/rules/ — リポジトリ単位の規約と値の追加
always/ プロジェクト内で常時注入
├─ project-context/rule.md 概要・技術スタック
├─ naming-values.txt 委譲受け口
├─ prh.yml 委譲受け口
└─ domain/dictionary/ ドメイン用語辞書
scoped/ 対象パス操作時のみ注入
├─ flow/approved-sha/ マージ時のみ
├─ project/context-scope/ コード編集時
└─ review-checklist/ レビュー観点の追加
✕ hook（機械強制）と違反時手順はグローバルが正本。プロジェクトで追加できるのは「値」のみ（語彙・命名・レビュー観点）
第1軸:

所有レベル — グローバル（共通）か プロジェクト固有（リポジトリ単位）か

第2軸:

注入タイミング — always（常時）か scoped（条件付き）か。両レベル共通の考え方
