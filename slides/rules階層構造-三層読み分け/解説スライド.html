CLAUDE CODE 設定管理

rules は scope → 分野 → 規約名の 3 階層ツリーで管理する

scope が注入タイミング（常時 or 条件付き）、分野と規約名がグルーピング。規約フォルダに rule.md と hook を同居させて機械強制する

.claude/rules/
├─ always/ 常時注入（eager）
│ ├─ agent/ 分野
│ │ └─ subagent-selection/ 規約名
│ │ ├─ rule.md 規約本文（paths なし → 全セッションで注入）
│ │ ├─ check-subagent-choice.sh hook 同居
│ │ └─ design-notes.txt 設計判断（非注入サイドカー）
│ ├─ naming/commit-branch/
│ │ ├─ rule.md + check-git-author-*.sh 同構造（規約 + hook）
│ │ └─ naming-values.txt 委譲受け口
│ └─ review-checklist/text-dictionary/
│ ├─ rule.md
│ └─ prh.yml 委譲受け口
├─ scoped/ 条件注入（lazy）
│ ├─ dev-flow/gate/
│ │ └─ rule.md paths: ["**/*.ts", "**/*.tsx"] → 対象ファイル操作時のみ注入
│ └─ review-checklist/code/common/
│ └─ rule.md paths: ["src/**"] → コードレビュー時のみ注入
└─ <repo>/.claude/rules/ プロジェクト委譲
└─ always/
├─ project-context/rule.md プロジェクト概要・ルート許可ディレクトリ
└─ text-dictionary/prh.yml 固有語彙の追加（枠組みの上書きは不可）
1

scope— always / scoped で注入タイミングを分ける

2

分野 / 規約名— 2 階層のフォルダでグルーピング

3

hook 同居 + 委譲— rule.md と check-*.sh を同フォルダに配置。値はプロジェクトへ委譲
