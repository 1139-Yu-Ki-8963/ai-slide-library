CLAUDE CODE 設定管理

rules は「常時」「条件」「委譲」の 3 層で必要な規約だけを注入する

常時注入 19 規約 + 条件注入 18 規約をセッション状況に応じて読み分け、プロジェクトは値の追加だけを委譲される

1

always（常時注入・eager）— 19 規約・10 分野

rules/always/<分野>/<規約名>/rule.md の階層で管理。paths 指定なし、全セッションに全件注入される。hook script（.sh）を同フォルダに同居させ、違反を block / advisory で機械強制する

agent/委任規約 naming/命名規約 response/応答品質 placement/配置ガード review-checklist/語彙辞書
always に加えて、触ったファイルのパスが paths 条件に一致すると追加注入される
2

scoped（条件注入・lazy）— 18 規約・6 分野

rules/scoped/<分野>/<規約名>/rule.md で管理。frontmatter の paths に一致するファイルを操作した時だけ注入される。全件を常時読み込まないことでトークン消費と注意力の分散を防ぐ

dev-flow/gate/ agent-config/hooks/ review-checklist/code/ review-checklist/document/
上位規約が「委譲可」を宣言している場合、プロジェクト側が受け口で値を追加できる
3

project 委譲（プロジェクト受け口）

<repo>/.claude/rules/ に配置。グローバル規約の「値」を追加する受け口。枠組み（hook の block/advisory 区分・違反時手順）はグローバルが常に正。値の削除・上書きは不可

prh.yml（プロジェクト固有語彙） naming-values.txt（コミット prefix） flow-values.yml（開発フロー設定）
1

always = 常時注入— 委任・命名・品質など全タスク共通の制約

2

scoped = 条件注入— ゲート・レビュー基準など文脈限定の制約

3

project = 値の委譲— 語彙・命名値・フロー設定の追加のみ
