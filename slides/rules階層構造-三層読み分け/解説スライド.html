CLAUDE CODE 設定管理

rules のフォルダ階層で「常時」「条件」「委譲」を管理する

.claude/rules/ の下に scope → 分野 → 規約名の 3 階層。規約フォルダに hook を同居させて機械強制する

1

always/ — 常時注入

paths なし。全セッションで無条件に注入

rules/always/
├─ <分野>/ 分野
│  └─ <規約名>/ 規約名
│      ├─ rule.md 規約本文
│      ├─ check-*.sh hook 同居
│      └─ design-notes.txt 非注入
✓rule.md と hook を同フォルダに配置
✓naming-values.txt 等で委譲受け口を提供
2

scoped/ — 条件注入

frontmatter の paths 指定に一致する時だけ注入

rules/scoped/
├─ dev-flow/gate/
│  └─ rule.md paths: ["**/*.ts"]
└─ review-checklist/code/
    └─ rule.md paths: ["src/**"]
✓不要な規約を読み込まずコスト抑制
✓対象ファイル操作時に高 attention で注入
3

<repo> — プロジェクト委譲

グローバル規約の値をプロジェクトで追加する受け口

<repo>/.claude/rules/
└─ always/
    ├─ project-context/rule.md 概要
    ├─ naming-values.txt 値の追加
    └─ prh.yml 値の追加
✓語彙・命名値をプロジェクトで追加
✕枠組み（hook・違反時手順）の上書き不可
1

always = 常時注入— 共通制約を漏れなく適用

2

scoped = 条件注入— 文脈に合う規約だけ読み込み

3

委譲 = 値の追加— 固有値を受け口で追加
