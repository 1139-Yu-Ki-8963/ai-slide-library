CLAUDE CODE 設定管理

規約は 3 フォルダに分けると AI が必要な分だけ読み込む

命名・レビュー基準は always/、場面限定は scoped/、プロジェクト固有値は受け口へ。hook 同居で機械強制する

1

always/ — 全タスク共通

どのタスクでも守るべき規約を置く。全セッションで自動注入される

rules/always/
├─ naming/commit-branch/ 命名規約
│  └─ rule.md + check-*.sh hook 同居
├─ review-checklist/text-dictionary/
│  └─ rule.md + prh.yml 語彙辞書
└─ agent/subagent-selection/
    └─ rule.md + check-*.sh 委任規約
✓rule.md と hook を同フォルダに配置
✓全セッションで自動注入される
2

scoped/ — 場面限定

特定のファイルを触った時だけ必要な規約を置く

rules/scoped/
├─ review-checklist/code/
│  └─ rule.md paths: ["**/*.ts"] コードレビュー基準
└─ dev-flow/gate/
    └─ rule.md paths: ["src/**"] 実装ゲート
✓不要な規約を読み込まずコスト抑制
✓対象ファイル操作時だけ注入
3

<repo> — プロジェクト固有

グローバル規約の値をプロジェクトごとに追加する受け口

<repo>/.claude/rules/
└─ always/
    ├─ project-context/rule.md 概要
    ├─ naming-values.txt 値の追加
    └─ prh.yml 値の追加
✓語彙・命名値をプロジェクトで追加
✕hook・違反時手順の上書きは不可
1

always = 常時注入— 命名・レビュー等の共通規約

2

scoped = 条件注入— コードレビュー等の場面限定

3

受け口 = 値の追加— 固有の語彙・命名値を追加
