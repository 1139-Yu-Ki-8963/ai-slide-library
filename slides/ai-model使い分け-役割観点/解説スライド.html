Claude Code 主力ツール

判断の重さに応じて 3 つのモデルを使い分ける。

Opus 高い判断力が必要な場面に限定して投入 設計判断・計画立案
Sonnet 大部分の開発作業をこなす汎用主力 コード修正・調査・レビュー
Haiku 判断不要の定型作業を高速に処理 コマンド実行・テスト起動
リミット到達で切り替え
Composer モデル 推奨

設計書を正しく読み、指示に沿った出力を返す。リミット到達時の代替として品質を維持できる。

✓ 設計書・長文コンテキストを最後まで読む
auto モード 非推奨

コスト効率を優先して低性能モデルが選ばれるため、品質に問題が出る。

✕ 設計書を最後まで読まないなど、指示の遵守に問題が生じる
